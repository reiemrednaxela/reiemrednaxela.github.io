Introduction

Demand Forecasting 📈

  • Crucial for inventory optimization, resource allocation, strategic planning

  • Often sales data is the best proxy for demand

    • It is censored by stockouts and other constraints

  • This gap between sales and demand leads to biased historical data and underestimated forecasts

Problem Formulation

  • Scenario 1: \[ \text{sales}_t = \text{demand}_t \times \text{censor}_t, \quad \text{where} \quad \text{censor}_t \in [0, 1] \]

  • Scenario 2: \[ \text{sales}_t = \min(\text{demand}_t, \text{cap}), \quad \text{for some cap} > 0 \]

Approach 2

Tobit censored regression 🤓

  1. Define loss (negative log-likelihood)
import jax.numpy as jnp
from jax.scipy.stats import norm

def tobit_negative_log_likelihood_lower(y_true, y_pred, sigma, L):
    # c.f. https://en.wikipedia.org/wiki/Tobit_model
    return -jnp.where(
        y_true == L,
        norm.logcdf((L - y_pred) / sigma),
        norm.logpdf((y_true - y_pred) / sigma) - jnp.log(sigma),
    )
  1. Let JAX calculate gradient and hessian
from jax import grad, hessian, vmap

def tobit_loss_upper(y_true, y_pred, U, sigma):
    grad_fn = vmap(
        grad(tobit_negative_log_likelihood_upper, argnums=1), in_axes=(0, 0, None, None)
    )
    hess_fn = vmap(
        hessian(tobit_negative_log_likelihood_upper, argnums=1), in_axes=(0, 0, None, None),
    )
    return grad_fn(y_true, y_pred, sigma, U), hess_fn(y_true, y_pred, sigma, U)
  1. Use the loss function in a model
get_ml_forecast(
    models=[
        LGBMRegressor(
            objective=tobit_loss,
            random_state=42,
        )
    ]
)

Practical Considerations

  • Simulation study to understand problem and different approaches
  • Not a silver bullet: The outcome is only as representative as your data generating process
  • There are also a lot of business specific solutions
    • Example: Hierarchical imputation in [1]

[1] Deep Learning based Forecasting: a case study from the online fashion industry

Evaluation Strategies

  • Be cautious with classical error metrics in censored situations
    • Consider the impact of your decision [1]
  • Possible strategies to handle censored data points:
    • Exclude censored data points
    • Impute missing values
    • Down-weight censored data points
  • Avoid using Mean Absolute Error (MAE) if you have a lot of zeros [2]

[1] What about all those zeroes? Measuring performance of models on intermittent demand

[2] Don’t use MAE based error measures for intermittent demand
